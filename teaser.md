# 1. Три сигма: Главное за одну страницу

Мы построили прототип раннего выявления студентов с риском отчисления по LMS-данным. Результат переведен в формат "Светофора": зеленая, желтая и красная зоны риска, чтобы тьютор видел, кого обрабатывать первым и почему.

Полный отчет: [final_report.pdf](final_report.pdf), Markdown-версия: [final_report.md](final_report.md).  
Основной код и финальное исследование: [Tri_sigma_final_research.ipynb](https://github.com/mariyakukhtinova-sys/hackaton_mipt_ds_25/blob/master/Tri_sigma_final_research.ipynb).

Ключевой результат на M3, midpoint:

1. Модель обработала 1407 активных студентов.
2. Красная зона риска - 211 студентов, то есть 15% потока.
3. Это сокращает первичную ручную проверку для тьютора с 1407 студентов до приоритетного списка из 211.
4. Рекомендуемая точка активного вмешательства - 14-й день модуля: уже видна большая часть активной популяции, но до резкого ухудшения красной зоны еще остается время.

# 2. Что сделано

1. Выбрана гранулярность `user-course`: один студент в одном модуле/курсе. Это важно, потому что один и тот же пользователь может проходить разные модули как разные учебные контексты.
2. Для M1 и M2 использован фактический статус. Для M3, где фактического статуса в данных нет, таргет получен улучшенной эвристикой по прохождению промежуточной аттестации.
3. Собрана temporal-витрина с временными отсечками `7_days`, `14_days`, `21_days`, `midpoint`. В каждый срез попадают только события до даты прогноза, поэтому модель защищена от утечки будущих данных.
4. Построены модели риска: Random Survival Forest (RSF) для ранжирования риска, CatBoost для классификационной проверки и модель Кокса для интерпретации факторов через Hazard Ratio.
5. Результат упакован в прототип "Светофор" с зонами риска и практическими действиями для тьютора.

# 3. Основные метрики

1. RSF: C-index = 0.7662 ± 0.0242. C-index показывает качество ранжирования риска: насколько правильно модель упорядочивает студентов по вероятности более раннего отчисления.
2. Out-of-Time проверка M2 -> M3: C-index = 0.7636. Это основная проверка переносимости модели на следующий учебный поток.
3. CatBoost на внутренней проверке M1+M2: ROC-AUC = 0.9076, recall риска = 0.83, precision риска = 0.45.
4. CatBoost на M3, обучение только на M2: AUC = 0.7416. Этот вариант оказался лучше для оперативного "Светофора", чем обучение на M1+M2, потому что последний завершенный модуль ближе к текущему контексту.

# 4. Какие паттерны нашли

Главный вывод: риск отчисления определяется не только тем, "много" или "мало" студент делает, а тем, насколько его поведение отклоняется от нормального ритма группы и курса.

Ключевые сигналы риска:

1. Длинная пауза с последнего действия относительно группы.
2. Концентрация учебы на выходных.
3. Аномальный всплеск кликов, который может означать попытку догнать курс, стресс или проблемы с навигацией.
4. Низкий охват уроков: студент кликает, но не движется широко по программе.

Защитные факторы:

1. Темп на уровне группы.
2. Широкий охват уроков.
3. Регулярные практические взаимодействия и ответы в LMS.
4. Посещение трансляций и занятий.

# 5. Интерпретируемость модели

Для интерпретации использованы три уровня анализа:

1. Spearman-корреляции по `stats__module_*` на M1-M2, где есть фактический статус.
2. Mutual Information на сырых LMS-событиях, чтобы увидеть нелинейные связи.
3. Permutation Importance и Cox Hazard Ratios, чтобы объяснить, какие признаки сильнее всего двигают риск вверх или вниз.

Это закрывает требование интерпретируемости: результат модели не остается "черным ящиком", а переводится в понятные поведенческие паттерны.

# 6. Прототип "Светофор"

На M3, midpoint модель делит студентов на три зоны:

1. Зеленая зона - 844 студента, 60% потока, низкий приоритет ручного вмешательства.
2. Желтая зона - 352 студента, 25% потока, профилактическое внимание.
3. Красная зона - 211 студентов, 15% потока, приоритетная ручная работа тьютора.

Практический смысл: вместо просмотра всего потока тьютор получает короткую очередь высокого риска и объяснение причин риска.

# 7. Когда вмешиваться

"Светофор" проверен на срезах 7, 14, 21 день и `midpoint`. Лучший старт активного вмешательства - 14-й день:

1. В мониторинге уже 1177 студентов, это 83% финальной активной midpoint-популяции.
2. Красная зона уже сформирована: 177 студентов.
3. До резкого ухудшения красной зоны на 21-й день остается около недели.
4. Нагрузка на тьюторов уже прогнозируема.

# 8. Соответствие критериям

1. Интерпретируемость паттернов: есть MI, Permutation Importance, Cox Hazard Ratios и бизнес-интерпретация ключевых факторов.
2. Корреляции и причинные гипотезы: посчитаны Spearman-корреляции, нелинейные связи и сформулированы гипотезы о механизмах риска без заявления доказанной причинности.
3. Соответствие задаче: есть прогноз риска и группы риска; основная переносимая метрика - OOT C-index 0.7636 на M2 -> M3.
4. Техническая завершенность: собраны витрины, temporal-срезы без утечки данных, сохранены артефакты и финальный ноутбук.
5. Прототип: реализован "Светофор" с зонами риска и приоритетом действий.
6. Наглядность: результаты представлены через ключевые графики, метрики и практический сценарий для тьютора.
